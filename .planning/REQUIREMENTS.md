# Requirements: Rent Contract Bot

**Defined:** 2026-03-24
**Core Value:** Полный цикл создания договора аренды за 2-3 минуты вместо 25-40 минут ручной работы

## v1 Requirements

### Dialog Flow

- [ ] **DIAL-01**: Пользователь может выбрать группу объектов (Г39 или Г38) через inline-клавиатуру
- [ ] **DIAL-02**: Пользователь может выбрать номер квартиры из списка для выбранной группы
- [ ] **DIAL-03**: Пользователь может ввести дату договора в формате ДД.ММ.ГГГГ
- [ ] **DIAL-04**: Пользователь может ввести дату Акта приёма-передачи в формате ДД.ММ.ГГГГ
- [ ] **DIAL-05**: Пользователь может ввести сумму ежемесячной аренды
- [ ] **DIAL-06**: Пользователь может ввести сумму депозита
- [ ] **DIAL-07**: Пользователь может выбрать способ внесения депозита (единовременно или 50%+50%)
- [ ] **DIAL-08**: Пользователь может ввести телефон арендатора в формате +7 XXX XXX XX XX
- [ ] **DIAL-09**: Пользователь может ввести email арендатора

### Validation

- [ ] **VALD-01**: Система валидирует формат даты (ДД.ММ.ГГГГ) и корректность значения
- [ ] **VALD-02**: Система валидирует формат телефона (+7 и 10 цифр)
- [ ] **VALD-03**: Система валидирует формат email
- [ ] **VALD-04**: Система валидирует денежные суммы (положительные числа)
- [ ] **VALD-05**: Система проверяет возраст арендатора (18+ лет на дату договора)
- [ ] **VALD-06**: При ошибке валидации пользователь получает понятное сообщение и может ввести данные повторно

### OCR

- [ ] **OCR-01**: Пользователь может загрузить фото первой страницы паспорта (основные данные)
- [ ] **OCR-02**: Пользователь может загрузить фото страницы с пропиской
- [ ] **OCR-03**: Система распознаёт паспортные данные через Claude Vision API (ФИО, дата рождения, место рождения, серия, номер, дата выдачи, кем выдан, код подразделения, адрес регистрации, пол)
- [ ] **OCR-04**: Система показывает распознанные данные для проверки перед генерацией договора
- [ ] **OCR-05**: Пользователь может подтвердить или отклонить распознанные данные

### Document Generation

- [ ] **DOC-01**: Система генерирует номер договора в формате группа/квартира/дата
- [ ] **DOC-02**: Система заполняет DOCX-шаблон данными из диалога и OCR
- [ ] **DOC-03**: Система конвертирует заполненный DOCX в PDF через LibreOffice headless
- [ ] **DOC-04**: Система отправляет готовый PDF в Telegram-чат как документ

### Database

- [ ] **DB-01**: Система сохраняет данные каждого договора в SQLite
- [ ] **DB-02**: Запись содержит все поля: номер договора, группа, квартира, данные арендатора, даты, суммы, путь к PDF

### Infrastructure

- [ ] **INFR-01**: Бот запускается командой `python main.py` без ошибок
- [ ] **INFR-02**: Команда /start начинает диалог создания договора
- [ ] **INFR-03**: Команда /cancel отменяет создание на любом этапе
- [ ] **INFR-04**: Критические операции (OCR, PDF, DB) логируются
- [ ] **INFR-05**: Ошибки в критических операциях перехватываются и пользователь получает понятное сообщение

## v2 Requirements

### Admin Features

- **ADMN-01**: Команда /history показывает историю созданных договоров
- **ADMN-02**: Команда /stats показывает статистику (всего договоров, активных)
- **ADMN-03**: Улучшенная обработка edge cases (дублирование номеров, таймауты)

### UX Improvements

- **UX-01**: Возможность редактирования отдельных полей перед подтверждением
- **UX-02**: Прогресс-индикатор для длительных операций (OCR, PDF)

## Out of Scope

| Feature | Reason |
|---------|--------|
| E-signature | Подписание происходит лично, добавляет сложность без пользы |
| OAuth / авторизация | Бот для личного использования |
| Мультиязычность | Только русский язык |
| Мобильное приложение | Telegram достаточно |
| Real-time уведомления | Не нужно для текущего use case |
| Редактирование договоров | Проще создать заново |
| CRM / мульти-пользователь | Один арендодатель, 15 квартир |
